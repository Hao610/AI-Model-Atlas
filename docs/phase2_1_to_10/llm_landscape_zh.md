## 🌳 大模型演进之树

## 🆚 闭源商业 API vs. 开源自部署权重

在构建 AI 应用时，你需要做的第一个重大架构决策是：**我们是使用云端的商业 API，还是自己部署开源模型？**

| 维度 | 闭源商业 API (如 OpenAI, Claude) | 开源/开放权重模型 (如 Llama, Qwen, DeepSeek) |
| :--- | :--- | :--- |
| **接入难度** | ⚡ **极易**：注册账号拿个 Key，写三行 Python 代码就能用。 | 🛠️ **中等**：需要搭建显卡环境，安装 Ollama 或 vLLM 等引擎。 |
| **数据隐私** | ⚠️ **存在风险**：数据需要通过互联网发送给第三方处理。 | 🔒 **极高**：完全部署在本地服务器上，数据绝不出网。 |
| **可控与定制**| ⚠️ **受限**：只能调整提示词，或者做基础的云端微调。 | 🎯 **无限**：可以完全修改模型参数、进行深度微调。 |
| **收费模式** | 💳 **按量付费**：按每次对话的 Token 数量计费，用户多则费用高。 | 🖥️ **设备成本**：一次性买断 GPU 显卡，后续使用完全免费。 |

## 📏 读懂模型大小（参数量）

开源模型在发布时，名字里通常会带有一个数字（例如 `Llama-3-8B`，`Qwen-72B`），这个 **B** 代表 **Billion (十亿参数)**。你可以这样来评估它们的实力：

### 1. 轻量级模型 (1B ~ 9B 参数)
* **代表模型**：`Llama-3-8B`、`Qwen-2.5-7B`、`Gemma-2-9B`。
* **硬件要求**：普通的家用笔记本电脑、手机、或最便宜的云服务器。
* **擅长任务**：文本分类、单篇短文总结、简单客服对话、将非结构化文本整理成 JSON 格式。

### 2. 中量级模型 (14B ~ 32B 参数)
* **代表模型**：`Qwen-2.5-14B`、`DeepSeek-Lite-16B`。
* **硬件要求**：高端游戏本、家用台式机（配 RTX 3090/4090）或中等显卡服务器。
* **擅长任务**：日常代码编写、较为复杂的逻辑推理、中英双语精准翻译。

### 3. 重量级模型 (70B 及以上参数)
* **代表模型**：`Llama-3-70B`、`Qwen-2.5-72B`、`DeepSeek-V3`（671B 混合专家模型）。
* **硬件要求**：企业级 GPU 服务器（如 Mac Studio 192GB 内存版，或多张 A100/H100 显卡）。
* **擅长任务**：接近人类水平的逻辑推导、高难度代码编写、多步骤复杂任务规划与 Agent 联动。
