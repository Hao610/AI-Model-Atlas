# 无代码 Agent 搭建指南 🛠️

[[English] (09_no_code_agents.md)](09_no_code_agents.md) | [中文]

你不需要写一行代码，就能做出功能强大的 AI 应用程序。现在的可视化开发平台如 **Dify** 和 **Coze (扣子)**（字节跳动出品）允许你通过直观的图形化界面，拼装出完全自主运行的 **AI Agent (智能体)**。

本章将带你解构一个无代码 Agent 的内部原理。

---

## 🧩 Agent 的五脏六腑

一个真正的 Agent 不仅仅是一个聊天框。它是以大语言模型 (LLM) 为“大脑”，并外接了以下三个关键能力的综合体：

```text
               ┌──────────────────────┐
               │    大语言模型 (LLM)   │
               │      (逻辑大脑)      │
               └──────────┬───────────┘
                          │
         ┌────────────────┼────────────────┐
         ▼                ▼                ▼
     [ 记忆库 ]        [ 工具箱 ]       [ 知识库 ]
   (记住聊天上下文)  (联网搜索、发邮件)  (上传的内部文档)
```

1. **设定 Prompt (人设与灵魂)**：规定它的角色定位、说话风格和行为规范。
2. **工具 (手和脚)**：允许大模型与现实世界交互的插件（例如：百度/谷歌搜索、发送邮件、查询天气、执行特定 API 等）。
3. **长期记忆 (大脑皮层)**：存储过往聊天历史，使 Agent 能在几天后依然记得你是谁。
4. **知识库 (私人图书馆)**：静态文件（如 PDF、Word）或数据库。模型在回答问题时会先去这里查阅，避免胡说八道。

---

## 🆚 Dify vs. Coze (扣子)

对于无代码开发者来说，这是目前市面上最好用的两个平台：

| 维度 | Dify | Coze (扣子) |
| :--- | :--- | :--- |
| **部署方式** | 开源（支持本地部署，也有云端版）。 | 完全托管的云端服务。 |
| **最佳用途** | 企业级内部数据流打通、本地数据库对接。 | 快速发布到社交平台（微信、飞书、Discord、Telegram）。 |
| **插件生态** | 偏向于技术和开发者级插件。 | 极度繁荣，拥有成百上千个开箱即用的生活、搜索和多媒体插件。 |
| **工作流引擎** | 极其严谨，非常适合设计结构复杂的逻辑流图。 | 界面极为友好，自带简单易用的代码沙箱。 |

## 📝 动手实操：如何设计一个客服 Agent

以下是在 Dify 上配置一个“企业客服助理”的配置蓝图：

1. **新建应用**：选择 **Chatflow (工作流)**（适合逻辑严密的工作）或 **Agent (智能体)**（适合灵活调用工具的场景）。
2. **设定核心提示词 (Prompt)**：
   > *“你是一家潮鞋网店的资深客服。请根据上传的『鞋款尺码与退换货规则』知识库回答用户的问题。如果遇到知识库里没有的信息，切勿胡乱编造，请触发『邮件发送工具』，将用户的问题和邮箱收集起来发送给人工客服。”*
3. **关联知识库**：上传你们公司的退换货 FAQ PDF。Dify 会自动帮你将 PDF 切片并做索引。
4. **绑定工具**：添加“谷歌搜索”工具和“邮件通知”插件。
5. **发布上线**：点击发布后，Dify 会直接给你生成一个网页链接。你把链接发给任何人，他们就能直接使用你刚刚做好的客服机器人。
