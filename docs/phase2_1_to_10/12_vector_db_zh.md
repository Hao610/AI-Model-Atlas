# 向量数据库入门 🗄️

[[English] (12_vector_db.md)](12_vector_db.md) | [中文]

在传统数据库（如 MySQL 或 Excel 表格）中，你只能通过**精确匹配**来查找数据（例如：搜索商品 ID `105`，或者精确搜索单词 `"苹果"`）。

但是在 AI 的世界里，我们需要进行**语义（含义）搜索**。例如，用户搜索 *“小猫”*，系统应该能自动帮你找出包含 *“喵星人”* 或 *“萌宠幼崽”* 的文档，即使这些文档里压根没有出现“小猫”这两个字。

为了实现这个功能，我们需要把文字转化为代表语义的“坐标”（向量），并把它们存放在专门的**向量数据库 (Vector Database)** 中。

---

## 📍 核心比喻：三维语义地图

想象一张巨大的三维立体地图，地图上的点代表不同词汇或段落的意思：

* **“国王”** 和 **“女王”** 因为意思相近，在地图上挨得很紧。
* **“苹果”** 和 **“香蕉”** 被归类在水果区，飘在另一个角落。
* **“电脑”** 和 **“笔记本”** 则堆在科技区。

向量数据库不看文字是怎么拼写的，它只计算**坐标之间的物理距离**。当你搜索 *“手机”* 时，它会查看“手机”在地图上的坐标，然后把周围离它最近的坐标拉出来（例如 *“智能机”*、*“移动设备”*）。

## 🏆 四大主流向量数据库对比

当你开始动手搭建 RAG 知识库系统时，通常会在这四款主流工具中做出选择：

| 数据库 | 学习难度 | 扩展性 | 最佳适用场景 |
| :--- | :--- | :--- | :--- |
| **Chroma** | ★ | ★★ | **初学者/快速原型开发**：完全运行在 Python 脚本的内存中，零安装成本，非常适合本地轻量化应用。 |
| **FAISS** (脸书开源) | ★★ | ★★★ | **纯本地高速检索**：一个底层的算法库（非完整数据库系统），专为单机环境下超高速度的数学计算而生。 |
| **Milvus / Qdrant** | ★★★★ | ★★★★★ | **企业级海量数据**：专业的分布式向量数据库，能轻松处理数亿级向量在集群节点间的分布式存储和检索。 |
| **PGVector** | ★★★ | ★★★★ | **混合型关系业务**：PostgreSQL 数据库的插件。如果你想把传统数据表和向量数据存在同一个库里，这是首选。 |

---

## 💡 新手选型指南

1. **如果你今天刚开始做 Demo 练习**：毫不犹豫选择 **Chroma**。它只需要一行 Python 代码初始化，不需要在电脑上安装任何繁琐的数据库软件。
2. **如果你的系统已经在使用 PostgreSQL 关系型数据库**：直接开启 `pgvector` 插件。这样可以避免为了引入向量搜索而额外维护一套新的数据库，让系统架构保持极简。
3. **如果你要为大型企业开发支撑高并发的知识库**：选择 **Milvus** 或 **Qdrant**，它们在生产环境中的性能和扩展性最为优秀。
