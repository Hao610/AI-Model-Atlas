# RAG 知识库检索 📚

[[English] (11_rag_intro.md)](11_rag_intro.md) | [中文]

如果你去问通用大模型你们公司的内部产品规划，或者你个人的私人日程安排，它要么会胡编乱造，要么会直接告诉你它不知道。这是因为模型在训练时并没有接触到这些私有的、实时更新的信息。

那么，如何在不重新训练模型的情况下，让 AI 能够读取并回答我们的私有数据呢？答案就是 **RAG (Retrieval-Augmented Generation，检索增强生成)**。

---

## 📖 生活比喻：闭卷考试 vs. 开卷考试

理解 RAG 最简单的方法，就是想象你正在参加一场历史考试：

* **传统大模型（闭卷考试）**：你坐在一间没有窗户、没有手机、没有任何书籍的考场里。你必须完全依赖大脑在之前背下来的知识。如果遇到极其冷门的问题，你可能会记错或只能瞎编。
* **RAG 模式（开卷考试）**：你坐在考场里，拿到问题。在作答之前，一位动作极快的“图书管理员”会迅速冲进你的私人图书馆，找出最相关的几页资料，并整齐地平铺在你的书桌上。你可以参考这些资料写出依据更充分的答案，但仍需要确认检索到的内容是否真的相关。

```text
 用户问题
    │
    ▼
【图书管理员】 ──► 搜索私有图书馆 ──► 找出最相关的几页内容 (上下文)
    │
    ▼
【拼装提示词】 (用户问题 + 找出来的纸张内容) ──► 喂给大模型 ──► 输出 factual 事实答案
```

## ⚙️ RAG 的工作原理：分步解析

RAG 在底层分为两大阶段：**入库准备** 与 **实时检索**。

### 阶段一：建立你的私有图书馆 (Ingestion)
1. **加载文档**：你把你们公司的 PDF 常见问答、Word 手册或网页链接喂给 RAG 系统。
2. **文本切片 (Chunking)**：系统会自动把一本厚厚的文档，切割成一小段一小段的“段落卡片”（比如每段 500 字）。这样可以防止大模型的“办公桌”（上下文窗口）被撑爆。
3. **向量化 (Embedding)**：系统把每一段文字，转化为一串代表其“语义坐标”的浮点数数组（向量）。
4. **存入向量数据库**：将这些“语义坐标”分类存储在专门的数据库中。

### 阶段二：用户提问与实时生成 (Retrieval & Generation)
1. **输入问题**：用户提问：*“我们公司对于受损商品的退款政策是什么？”*
2. **检索 (Retrieval)**：系统把用户的问题也转化为语义坐标，去向量数据库里检索，瞬间拉取语义上最接近的 3 个切片段落。
3. **打包提示词**：系统把检索出来的段落和用户的问题组装在一起：
   > *“请仅根据以下参考资料回答用户问题。参考资料：[在此插入刚才检索出来的3段退款政策文字]。用户问题：我们公司对于受损商品的退款政策是什么？”*
4. **生成回答**：大语言模型（LLM）阅读被塞到嘴边的参考资料，生成依据更充分的回答；最终质量仍取决于检索结果、提示词和模型本身。

## ⚖️ RAG 检索 vs. 模型微调 (Fine-Tuning)

很多初学者容易混淆这两者的定位。以下是它们的直观对比：

| 维度 | RAG (开卷考试) | 模型微调 (大脑手术) |
| :--- | :--- | :--- |
| **主要目的** | 让模型获取最新的、私密的**事实性知识**。 | 调整模型的**说话风格、输出格式或特定语种能力**。 |
| **资金与时间成本**| 极低。 | 较高（需要高性能显卡，训练耗时）。 |
| **更新知识速度** | 瞬间完成（在数据库里上传或删除一个 PDF 即可）。 | 较慢（每次更新都需要重新跑一遍训练任务）。 |
| **回答事实准确度**| 检索内容相关时更有依据，但仍需要验证。 | 中等（模型依然可能出现幻觉）。 |
