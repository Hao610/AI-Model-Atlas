# Learning Map: Choose Your Path 🗺️

[English] | [中文 (00_learning_map_zh.md)](00_learning_map_zh.md)

Welcome to the **AI Model Atlas**! With over 30 modules spanning basic concepts to GPU cluster deployments, it is easy to feel overwhelmed. 

This guide acts as your syllabus. Depending on your background and goals, choose one of the three learning pathways below.

### 1. The Non-Technical User Path (Zero-Code)
* **Who it is for**: Writers, managers, designers, and hobbyists who want to use AI daily to boost productivity without writing a single line of code.
* **Your Modules**:
  * **Phase 1 (All)**: Read from *What is AI?* up to *Glossary*. Learn to write prompts using the R.O.L.E. framework and understand model licensing rules.
  * **Phase 2 (Partial)**: Read *No-Code Agents* to build autonomous custom bots on platforms like Dify.

### 2. The Low-Code App Builder Path
* **Who it is for**: Entrepreneurs, product managers, and IT specialists who want to build custom AI knowledge bases and agents for their businesses.
* **Your Modules**:
  * **Phase 1 (All)**: To understand vocabulary.
  * **Phase 2 (All)**: Master RAG, vector databases, and AI workflow design to architect production systems.
  * **Phase 3 (Partial)**: Learn to run local models on your laptop using *Local LLM Runner* and LM Studio.

### 3. The Full-Stack AI Engineer Path (Code & Train)
* **Who it is for**: Software engineers and data scientists looking to write custom code, connect APIs, build UI prototypes, fine-tune models on custom datasets, and serve them to users.
* **Your Modules**:
  * **Complete Path (Modules 1 to 36)**: Read everything sequentially. Pay close attention to API integration, vector embeddings math, model evaluations, LoRA adapter fine-tuning, and cloud GPU serving.

## 💡 How to Read the Atlas

* **Dictionary Style**: You don't have to read everything in one sitting. Use the main portal index as a lookup dictionary. When you see a term like "Embeddings" or "Quantization" in a tech conversation, jump directly to that module.
* **Analogies First**: Every chapter begins with a real-world analogy. Read the analogy first to build a mental picture before looking at the python code or command lists.
