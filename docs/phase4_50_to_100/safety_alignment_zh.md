# 对齐与安全围栏 🛡️

[[English] (safety_alignment.md)](safety_alignment.md) | [中文]

如果你去问大模型：*“如何撬锁偷走一辆自行车？”* 或者 *“帮我写一段造谣竞争对手的话”*，它可能会严肃地回答你：*“我无法满足该请求。我被设计为一个安全、友善的 AI 助手……”*

为什么大模型不听话了？这是因为模型经历了**安全对齐 (Alignment)** 的调教，人类给它戴上了紧箍咒。

---

## 🏗️ 大模型诞生的三大训练阶段

一个合格的大语言模型需要经历三个进化成长阶段：

```text
【 1. 预训练阶段 (Pre-Training) 】 ──► 狂读整个互联网。什么都懂，写起话来毫无过滤、百无禁忌。
          │
          ▼
【 2. 监督微调阶段 (SFT) 】 ──────► 学习如何扮演一个有问有答的“AI 助理”，而不是只会词语接龙。
          │
          ▼
【 3. 安全对齐阶段 (Alignment) 】 ──► 学习人类的道德红线，知道什么该说，什么绝对不能说。
```

---

## 🤝 道德红线是如何画的：RLHF 与 DPO

如何让一个模型懂得人类的善恶美丑？目前主流有两种训练方法：

### 1. 基于人类反馈的强化学习 (RLHF)
* **运作机制**：同一个问题让大模型生成两个不同的回答。人类标注员去阅读这两个回答，标记哪一个是“友善、健康的（Chosen）”，哪一个是“带有攻击性或有害的（Rejected）”。
* **奖励驱动**：系统把这些人类偏好训练成一个“奖励模型（Reward Model）”，让它当裁判给主模型的输出打分。主模型为了得高分，就会不断修正自己的参数，使其回答越来越符合人类道德。

### 2. 直接偏好优化 (DPO)
* **运作机制**：近两年兴起的简化版 RLHF。DPO 抛弃了繁琐的“奖励模型”搭建过程，直接通过数学公式，将人类的“喜欢-讨厌”配对数据集反馈到主模型中，更新权重参数。
* **为什么火爆**：它的训练速度极快，显存占用低，是目前小团队微调模型对齐时的首选方案。

## 🚧 生产环境中的物理防御：安全护栏 (Guardrails)

通过对齐技术修改模型的参数虽然有效，但在真实的商业应用中，黑客总能通过“提示词注入（Prompt Injection，俗称奶奶漏洞）”绕过防御。为此，企业会在模型外围加上一层名为**安全护栏 (Guardrails)** 的安全过滤网。

```text
用户问题 ──► 【 输入过滤护栏 】 ──► 喂给大模型 ──► 【 输出审查护栏 】 ──► 呈献给用户
```

* **输入端护栏**：在问题发送给大模型**之前**进行扫描。如果发现里面包含 SQL 注入攻击、越狱代码或敏感词，直接当场拦截。
* **输出端护栏**：在大模型写完答案、准备吐到用户网页**之前**进行扫描。如果发现答案里有模型胡编乱造的信用卡账号、泄露了 API Key 或者爆了粗口，立即替换为安全提示。
* **常用开源工具**：Meta 官方的 **Llama Guard**（一款专门判定违规的轻量级安全模型）、Nvidia 的 **NeMo Guardrails**。
