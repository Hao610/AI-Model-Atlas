# LoRA 极简原理解释 🎨

[[English] (25_lora_explained.md)](25_lora_explained.md) | [中文]

在过去，微调一个大语言模型需要修改它脑子里的全部参数。这被称为**全参数微调 (Full Fine-Tuning)**，需要成百上千张昂贵的显卡跑上几天，耗资数十万甚至上百万美元。

为了让普通的个人开发者和小企业也能玩得起微调，科学家们研发了一项改变游戏规则的技术：**LoRA (Low-Rank Adaptation，低秩适应)**。

---

## 🖼️ 核心比喻：Photoshop 的滤镜图层

想象一下，你手里有一张分辨率极高、体积巨大的高清原图（代表**基础模型**，如 Llama 3 权重文件）。你想调整它的色彩，让它呈现出一种复古胶片风格（你的**专属微调效果**）。

* **全参数微调 (Full Fine-Tuning)**：你直接打开原图，手动去修改图中的每一个像素点的颜色代码。修改完后，原图被覆盖。文件体积依然巨大，而且修改过程耗时耗力。
* **LoRA 微调**：你完全不碰这张底图。你在它上面新建了一个透明的**“滤镜图层”**（在 PS 中称为图层蒙版）。你只在这个滤镜图层上涂抹颜色修改。原图被完全冻结（不作修改）。当最后导出时，两层叠加在一起输出。

```text
  【 透明滤镜图层 (LoRA 适配器: 约 20MB) 】   <─── 我们只训练这个小图层！
                       │
                       ▼ (使用时实时叠加)
  【 高清原图底片 (Llama 3 基础权重: 约 16GB) 】 <─── 保持冻结，完全不改！
```

---

## ⚡ 为什么说 LoRA 是微调的救星？

因为我们冻结了庞大的基础模型，只训练极轻量的“滤镜图层”，这带来了极其恐怖的工程优势：

1. **显存要求暴降**：以前微调需要 8 张企业级 A100 显卡。而使用 LoRA，你只需要一张普通的家用游戏显卡（如 RTX 3090 / 4090）就能跑起来。
2. **文件体积暴缩**：一个完整的模型文件通常有 **15GB ~ 140GB**。而一个训练好的 LoRA 插件文件，通常只有 **10MB ~ 100MB** 左右。
3. **零成本热插拔切换**：由于基础模型是不变的，你可以在显存里只加载一次 Llama 3 基础模型，然后根据用户的请求瞬间切换不同的 LoRA 滤镜：
   * 叠加 *客服 LoRA* (10MB) ──► 变身网店客服。
   * 拔掉它，换上 *代码 LoRA* (15MB) ──► 变身写代码助手。
   * 拔掉它，换上 *合同法务 LoRA* (20MB) ──► 变身律师助手。

---

## 📏 到底什么是“Rank” (秩)?

当你去配置微调参数时，会遇到一个叫 **Rank (r)** 的参数设置（默认通常是 `8`，`16` 或 `64`）。
* **Rank** 决定了这个“滤镜图层”的**厚度（复杂度）**。
* **Rank 设得低（例如 8）**：代表图层很薄，占用的显存极少，训练速度飞快。但只能记录简单的语调和格式调整。
* **Rank 设得高（例如 64）**：代表图层能承载更复杂的计算，可以学到更深奥的专业领域逻辑，但代价是消耗更多显存，训练也变慢。
