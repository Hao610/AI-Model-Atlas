# 数据准备与清洗 📂

[[English] (23_data_preparation.md)](23_data_preparation.md) | [中文]

在机器学习领域，有一条至理名言：**“Garbage In, Garbage Out”（垃圾进，垃圾出）**。无论你的显卡有多贵、训练参数调得多完美，如果你喂给模型的数据是混乱、重复或错误的，最终微调出来的模型效果一定会惨不忍睹。

本章将为你讲解微调数据集的标准格式、清洗 checklist，以及如何利用大模型自动化生成微调所需的“合成数据”。

---

## 📄 1. 数据集标准格式：JSON vs. JSONL

大模型微调的数据集本质上是一组“对话历史”或“问答对”。最常见的是以下两种格式：

### JSON 格式 (标准数组)
适合小规模数据集（如 1000 条以下），结构是一个大数组包着对象。
```json
[
  {"instruction": "问题1", "input": "", "output": "答案1"},
  {"instruction": "问题2", "input": "", "output": "答案2"}
]
```

### JSONL 格式 (每行一个 JSON)
最适合中大型数据集。**JSONL (JSON Lines)** 指的是文件里的每一行都是一个独立的、合法的 JSON 对象。
```jsonl
{"instruction": "问题1", "input": "", "output": "答案1"}
{"instruction": "问题2", "input": "", "output": "答案2"}
```
*优势：Python 训练脚本在加载时，可以按行读取，不需要把一整个几 GB 大的 JSON 文件一次性全部塞进系统内存，能有效防止内存溢出。*

## 🧼 2. 数据清洗 Checklist (防爆显存与过拟合)

在把数据喂给 LLaMA-Factory 之前，请务必执行以下清洗步骤：

1. **去重 (Deduplication)**：删除完全重复的问答对。如果同一条数据反复出现，模型会对该段内容“死记硬背”（过拟合），丧失灵活性。
2. **格式标准化**：确保标点符号（中英文冒号、括号）、排版格式以及 System 提示词在所有样本中保持高度一致。
3. **过滤低质回复**：筛除掉类似 *“不知道”*、*“请查看说明书”*、*“无法回答”* 这样的废话回答。否则，微调后的模型会学会“偷懒”和推诿。
4. **控制 Token 长度**：检查每条数据转化为 Token 后的长度。确保没有任何一条样本超过了你设定的最大上下文长度（例如限制在 2000 个 Token 以内），防止在训练时因为某一条超长数据而导致 GPU 显存溢出报错 (OOM)。

## 🧬 3. 合成数据生成 (利用大模型造数据)

如果你的手头只有 100 页的公司内部产品手册 PDF，却一条“问答对数据”都没有，该怎么微调？你可以使用逻辑能力最强的商业模型（如 GPT-4o 或 Claude 3.5）作为“数据标注员”，帮你把这 100 页材料转化为 1000 条高质量的问答数据集。

你可以对大模型发送如下提示词模板：

> *“你是一位资深的数据标注专家。请仔细阅读以下产品手册段落，并针对该段内容，生成 5 个用户可能提出的真实问题，并附带最符合公司客服标准的详细回答。请严格按以下 JSON 格式输出，不要输出任何解释性文字：*
>
> `[{"instruction": "用户提出的问题", "input": "", "output": "标准客服回答"}]`
>
> *产品手册内容如下：[在此粘贴手册的第 1 页文字内容]”*

利用 Python 脚本循环遍历手册的每一页，在几分钟内花几块钱，你就能拥有一份包含 500 条高质量微调语料的专属数据集！
