# 🌊 第二章：非结构化攻击面（模型权重、中间件与数据流）

人工智能彻底打破了边界分明的传统安全防线。面对非结构化的数据输入和概率性的计算逻辑，大模型所处的是一个数据与代码界限完全模糊的混沌环境。

## 🚰 液体类比

* **类比 (The Analogy)**: 保护传统软件就像给房子的门窗上锁，而保护大模型就像试图约束流动的液体。
* **运作方式 (How it works)**: 传统系统中，你可以在明确的入口（如API）部署防火墙。而在AI中，海量的非结构化文本、音频和图像流直接绕过固定的控制点，涌入模型的核心逻辑。
* **关键概念 (Key Concept)**: 当攻击面随着上下文流体般地扩张和变化时，传统的边界防御将彻底失效。

## 📊 快速对比

| 概念 | 传统方式 | LLM 时代 | 影响 |
| --- | --- | --- | --- |
| **逻辑基础** | 硬编码的 `if/else` 规则 | 数十亿的概率性权重 | 无法确定性地预测系统行为。 |
| **输入类型** | 结构化的变量与类型 | 非结构化数据流（文本、音频） | 标准的输入校验工具完全失效。 |
| **边界定义** | 固定的入口点（API） | 动态的上下文窗口 | 只要有数据摄入的地方都可能发生攻击。 |
| **供应链** | 代码库与依赖包 | 庞大且不透明的模型权重 | 后门隐藏在数学矩阵中，而非代码里。 |

## 🧠 核心概念

1. **模型权重（模型劫持）：** 下载被污染的模型或恶意的 LoRA 适配器会绕过所有传统安全机制，将触发器深埋在不透明的数学海洋中。
2. **中间件（中毒的流水线）：** 攻击者将恶意载荷注入编排层和向量数据库，中间件随后会盲目地将其作为受信任的指令直接喂给模型。
3. **数据流（隐形上下文操纵）：** AI 不断摄取外部环境数据（解析 PDF、读取网页），攻击者可以在环境中埋下隐形陷阱（如零像素文本），无需直接交互即可劫持 AI。
