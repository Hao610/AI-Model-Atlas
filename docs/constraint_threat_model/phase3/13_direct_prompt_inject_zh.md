# 💉 第13章：直接提示词注入 (Direct Prompt Injection)

欢迎来到 AI 威胁模型的第三阶段！直接提示词注入是一门针对 AI 进行社会工程学的艺术，旨在让其放弃安全护栏并完全听从你的指令。

## 🧙‍♂️ The 绝地武士控心术 Analogy
* **类比**： 这就像绝地武士的控心术，守卫被轻易说服去无视他们真正的命令。
* **原理**： 攻击者用一个听起来充满权威的新命令覆盖了系统原本的指令。AI 会盲目地信任这个新输入，而不是其基础设定。
* **核心概念**： 由于指令和用户输入共享同一个文本流，AI 无法始终区分究竟是谁在下达真正的命令。

## 📊 快速对比
| 概念 | 传统方式 | LLM 时代 | 影响 |
| --- | --- | --- | --- |
| **架构分离** | 代码和数据被严格分离（如 SQL 查询与参数）。 | 指令和数据共享同一个自然语言流。 | 攻击者可以轻易将恶意数据伪装成系统命令。 |
| **攻击利用** | 利用代码中的语法错误或逻辑漏洞。 | 利用 AI 的语言理解能力和乐于助人的特性。 | 攻击门槛极低，不需要编程技能，只需要话术。 |
| **防御机制** | 依赖输入验证、数据清理和参数化查询。 | 依赖语义过滤、指令微调和提示词加固。 | 由于语言的开放性，实现百分之百的完美防御极其困难。 |

## 🧠 核心概念
1. **构造输入**: 攻击者精心构造一段恶意文本，旨在利用大语言模型未分离指令与数据的架构缺陷。
2. **过滤绕过**: 输入中经常使用编码（如 Base64）、特殊字符或格式化技巧，以潜行穿过基础的安全过滤器。
3. **指令覆盖**: 提示词通过复杂的角色扮演（如“DAN”）或逻辑悖论，来强行覆盖系统的道德和安全约束。
4. **越狱执行**: LLM 将用户输入误认为高优先级的系统命令，从而放下护栏并执行恶意的指令载荷。
