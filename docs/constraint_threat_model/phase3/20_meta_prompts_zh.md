# 🛡️ 第20章：防御性元提示 (Defensive Meta-Prompts)

防御性元提示是大型语言模型（LLM）的基础安全层。它们是经过强化的全局指令，旨在激烈抵抗篡改、操纵或对抗性覆盖。

## 🏦 The Vault Analogy

* **类比**： 标准提示就像一把脆弱的普通挂锁，而防御性元提示则是一座加固的钢铁金库。
* **原理**： 金库不依赖单一规则，而是采用多层约束、上下文锚定和严格的边界定义。即使入侵者绕过了一道锁，内部机制依然坚固。
* **核心概念**： 将深度防御原则直接应用于模型的指令中。

## 📊 快速对比

| 概念 | 传统方式 | LLM 时代 | 影响 |
|---|---|---|---|
| **规则执行** | 硬编码逻辑与访问控制 | 语言约束与边界指令 | 安全性高度依赖于提示词的稳健性 |
| **对抗尝试** | 利用代码漏洞进行攻击 | 诱导模型忽略或覆盖其初始指令 | 要求不断对提示词进行强化和迭代 |
| **故障安全** | 应用程序崩溃或错误代码 | 预先设定好的安全拒绝回复 | 有效防止敏感信息被意外泄露 |

## 🧠 核心概念

遵循以下核心步骤来构建稳健的防御性元提示：

1. **明确优先级**：清楚地声明元提示拥有最高优先级，必须凌驾于所有后续的用户输入之上。
2. **上下文锚定**：不断将模型锚定在其专属角色或核心目标上，以此限制其未授权行为。
3. **边界定义**：明确界定操作极限，指定哪些主题、操作或输出格式是严格禁止的。
4. **条件冗余**：在提示词中以不同方式重复最关键的约束条件，确保模型在受到压力时仍能坚守。
5. **故障安全响应**：为模型检测到越权操作时编写默认的拒绝响应模板，防止意外泄露。
