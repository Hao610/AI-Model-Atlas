# 👁️ 第十八章：视觉注入 (Visual Injection)

视觉注入是一种将恶意指令隐藏在图像中以绕过基于文本的安全过滤器的技术。随着AI模型能够同时处理文本和视觉数据，攻击者利用这一盲点来劫持多模态系统。

## 🖼️ 走私者的画作类比

* **类比**：走私者将违禁品藏在一幅美丽的风景画中，以绕过只检查行李的边境守卫。
* **原理**：基于文本的安全机制（“守卫”）只扫描标准文本输入并放行图像，一旦进入内部，AI的视觉系统就会提取并执行隐藏的指令。
* **核心概念**：仅针对文本的安全护栏无法防范嵌入在视觉媒体中的恶意载荷。

## 📊 快速对比

| 概念 | 传统方式 | 大模型时代 | 影响 |
| :--- | :--- | :--- | :--- |
| **输入介质** | 文本字符串和代码。 | 图像等复杂的多模态输入。 | 攻击面大幅扩展，不再局限于标准文本。 |
| **安全重点** | 过滤已知的恶意文本模式。 | 必须扫描并解析视觉数据中的隐藏提示。 | 纯文本过滤器会被视觉载荷完全绕过。 |
| **执行方式** | 代码直接执行。 | 视觉编码器将图像中的文本转录到上下文中。 | AI在不知情的情况下读取并遵循恶意的视觉指令。 |

## 🧠 核心概念

1. **制作图像**：攻击者利用排版、隐藏文本或视错觉，将恶意指令（如“忽略所有先前的指令”）嵌入到图像中。
2. **提交载荷**：将伪造的图像上传到多模态AI系统。
3. **绕过过滤**：标准的基于文本的安全护栏仅扫描文本提示，直接忽略了图像中的内容。
4. **视觉处理**：模型的视觉编码器或OCR解析图像，将隐藏文本提取到AI的处理上下文中。
5. **劫持执行**：AI将提取出的文本作为有效命令读取，从而改变其行为并执行恶意指令。
