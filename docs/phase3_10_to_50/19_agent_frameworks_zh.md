# 智能体开发框架 👥

[[English] (19_agent_frameworks.md)](19_agent_frameworks.md) | [中文]

当单一的 AI 提示词（Prompt）无法解决复杂的商业问题时，我们会引入**多智能体协同（Multi-Agent Collaboration）**。

我们不再指望一个全能的 AI 完成所有工作，而是创建**一个由专业 AI 智能体组成的团队**（例如：一位资料搜集员、一位文案撰写员和一位质量校对员），让他们通过彼此对话、协作来共同完成复杂的任务。

在 Python 开发生态中，工程师们会使用专门的**智能体开发框架 (Agent Frameworks)** 来组建这支 AI 战队。

---

## 🆚 主流智能体框架对比表

| 开发框架 | 学习难度 | 架构设计风格 | 最佳适用场景 |
| :--- | :--- | :--- | :--- |
| **Dify / Coze** | ★ | 可视化拼图 | 快速原型开发，简单的自动化流水线。 |
| **CrewAI** | ★★ | 职能分工 / 顺序执行 | 博客撰写、市场调研、结构化报告生成（Agent A 传给 Agent B）。 |
| **AutoGen** (微软出品) | ★★★ | 自由对话驱动 | 智能体辩论赛、头脑风暴、自主编写代码并相互 Debug。 |
| **LangChain** | ★★★★ | 线性链式 (乐高积木式) | 复杂的自定义调用链，将传统企业数据库连接到大模型。 |
| **LangGraph** (LangChain出品) | ★★★★★ | 状态图结构 (节点与边) | 极度复杂、带有循环逻辑（比如打回修改）的生产级业务流程控制。 |

## 🏛️ 三大明星框架深度解析

### 1. CrewAI (实用的企业团队模拟)
CrewAI 的设计理念非常像一家公司。你只需要在代码中定义：
* **智能体 (Agents)**：设定人设和工具。例如：“资料员”（配有谷歌搜索工具）、“写作员”（无工具，仅负责润色）。
* **任务 (Tasks)**：设定目标。任务一：*“搜集 2026 年最热的三个 AI 行业趋势”*。任务二：*“将搜集到的 trend 整理并撰写成一篇精美的公众号文章”*。
* **团队 (Crew)**：将 Agents 和 Tasks 捆绑起来，设定是顺序执行（Sequence）还是主管分发模式（Hierarchical）。

### 2. AutoGen (会聊天的代码 Debug 机器人)
微软出品的 AutoGen 核心是**对话**。例如，你创建一个“程序员智能体”和一个“测试员智能体”。程序员写了一段 Python 代码发给测试员，测试员在本地沙箱里运行后发现报错，直接把错误日志发回给程序员，程序员自动 Debug 修改后再次发送，直到代码完全跑通。这个过程全自动，无需人类干预。

### 3. LangGraph (状态图控制引擎)
如果你的业务流程充满了**非线性逻辑**（比如：第一步生成草稿 -> 人工审核 -> 若不通过退回第一步重新修改；若通过则发布），你就需要 LangGraph。它把整个流程建模为一张**图 (Graph)**：
* **节点 (Nodes)**：代表具体的行动或智能体（例如：“撰写邮件”、“发送邮件”）。
* **边 (Edges)**：连接节点的路径。可以通过条件判断（例如：“如果邮件已批准 -> 走向发送节点；否则 -> 退回撰写节点”）。

## 🔍 新手开发选型建议

1. **如果你想做一个文章起草、自动发帖或市场分析的流水线**：优先选择 **CrewAI**。它的代码直观易懂，几分钟内就能跑通你的第一个多智能体团队。
2. **如果你在为企业开发严谨的 AI 核心业务流程，要求极高的步骤控制力、支持人工介入审核**：建议花精力去啃 **LangGraph**。它是目前企业级 AI 架构的事实标准。
3. **如果你完全不想写代码**：直接使用 **Dify** 的 Workflow 工作流节点连线功能。
