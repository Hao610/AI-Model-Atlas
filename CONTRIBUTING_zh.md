# 社区贡献指南 🤝

首先，衷心感谢您对共建 **AI-Model-Atlas** (AI 模型图谱) 项目的关注！正是因为有开源社区开发者的支持，知识库才能不断迭代和完善。

当您想为本项目提交修改或贡献新内容时，请遵循以下规范：

---

## 🚦 如何参与贡献

### 1. 提交纠错与 Bug 修复
如果您在阅读中发现了错别字、代码报错、或者是失效的 Markdown 链接：
- 可以直接在 GitHub 上提交一个 **Issue** 进行反馈。
- 或者直接修改该文件，并提交一个 **Pull Request (PR)**。

### 2. 提议补充新的科普话题
如果您觉得我们遗漏了某个非常关键的 AI 模型、框架或开发概念：
1. 首先提交一个带有 `proposal` 标签的 **Issue** 说明您的想法。
2. 简要列出您想补充的新章节的目录大纲。
3. 经项目维护者沟通确认后，您可以按照中英文双语对照的格式编写好文档，并提交 PR。

---

## 🖋️ 文档编写风格规范

- **名师教学人设**：请继续保持以“专业导师”的视角撰写文档。多用生活比喻、对比表格和分步清单，避免使用枯燥堆砌的学术公式。
- **免截图原则**：禁止在文档中插入软件界面的截图。所有的 UI 和工作流请使用 Markdown 代码块、表格或 Mermaid 状态图进行手绘，以保证文档不会随着软件升级而过时。
- **双语并行原则**：如果您修改或新增了英文版文件 `[topic].md`，请务必同时补充或修改对应的中文版文件 `[topic]_zh.md`。
- **链接规范**：所有文件之间的跳转必须使用**相对路径**（例如 `../folder/file.md`），严禁使用绝对路径。
